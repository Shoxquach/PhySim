### 基本的物理模拟
对刚体进行物理模拟的核心代码在 Simulator.cpp 中，其中定义了一个类`Rigid`，用于存储一个刚体的质心位置，质量，惯性张量，速度，朝向，角动量等物理信息，同时，也采用显示积分方法实现了时间步物理模拟以及刚体碰撞模拟函数.
```c++
struct Rigid {
    float m, inv_m;
    glm::mat3 I, inv_I;
    glm::vec3 x;
    glm::vec3 v;
    glm::quat q;
    glm::vec3 L;
    glm::vec3 w;
    //...
}
```
定义长方体刚体对象，需要先指定质量`m`，位置`x`，朝向`q`，初速度`v`，初角动量`L`，并根据指定的长方体尺寸计算其自身坐标系下的惯性张量`I`，同时，也自动计算出质量与惯性张量的逆（`inv_m`，`inv_I`）. 特别地，对于被固定的刚体（例如固定的地面与墙面），只需令`inv_m`和`inv_I`为零. 在后面的计算中，将`q`转换为旋转矩阵`R`，从而将刚体对象自身坐标系下的惯性张量变换为世界坐标系中的惯性张量`I_world`.

`Rigid::SimulationTimestep`采用显示欧拉积分，对单个刚体对象，根据给定的外力和外力矩进行一定时间步的模拟. 模拟过程中，可以设定阻尼参数`damping`对速度和角动量进行自然衰减.

`collide_test`可以检测两个长方体刚体之间的碰撞，并使用基于冲量的方法计算碰撞后刚体的速度. 碰撞检测采用了 fcl 库，可以获取碰撞的位置，法向量和穿透深度信息. 当检测到碰撞点时，函数分别计算两个刚体碰撞点处的速度，进而得出相对速度，分解为法向分量和切向分量，结合恢复系数，摩擦系数计算碰撞后的速度. 根据相对速度的变化量，再根据两个刚体自身性质，计算碰撞的冲量，最终根据冲量来更新两个刚体的速度与角动量. 在对多个刚体场景的模拟中，依次对每一对刚体进行碰撞检测.
对于面-面碰撞这种检测存在多个碰撞点的碰撞情景，对每个碰撞点依次处理会导致处理不对称. 当前采用的处理是取所有碰撞点的平均位置，但是在两个略错开叠放的长方体刚体的情境中，会导致上方的刚体受到的力不通过质心而自动滑移.

### demo 功能
demo 中首先实现了三个基础的实例.

`CaseSingleRigid`中实现了对单个刚体进行物理模拟的 demo，可以通过键盘对刚体施加外力和外力矩.

`CaseTwoRigids`中实现了对两个刚体的碰撞进行物理模拟的 demo，默认给定了三种不同的碰撞场景，可以调节恢复系数来观察不同的碰撞效果，也可以通过键盘对刚体施加外力和外力矩.

`CaseFourRigids`中实现了四个长方体刚体的物理模拟 demo，场景还包含由固定刚体构成的地面和墙面，整体形成一个上方开口的长方体箱，并施加重力，从而实现较为复杂的物理模拟. 初始，四个刚体被设定于箱子上方不同的高度，且水平方向上的位置略有错开. 可以调节恢复系数，摩擦系数以及水平方向偏移程度等参数来观察不同的模拟效果，也可以通过键盘对其中的某个刚体施加外力和外力矩.

上述实例都支持鼠标交互来旋转或放缩摄像头，以及键盘交互对刚体施加外力，相对初始的摄像机角度，`→`，`←`，`↑`，`↓`，`,`，`.`按键分别对刚体施加向右，向左，向上，向下，向前，向后的固定外力，`a`，`d`，`s`，`w`，`q`，`e`按键分别对刚体施加使其从正面观察向左，向右，向上，向下，逆时针，顺时针旋转的固定力矩；在`CaseFourRigids`中，可以通过`~`键切换施加外力的刚体.

### 约束动力学实现
在`Simulation.cpp`中，还尝试实现了基于约束的物理过程模拟方法，来对`CaseFourRigids`中的相同场景进行模拟，相应实例为`CaseConstraintDynamics`，能够根据外力更新物体状态，并同时处理碰撞. 函数检测所有刚体之间的碰撞点，将碰撞中的参数转化为约束，通过矩阵运算解出每个碰撞点的冲量，再更新每个刚体的速度与角速度. 当前的约束动力学方法呈现的效果还不稳定，有时会出现物体快速被弹开，穿模以及数值爆炸的情形，尤其是恢复系数较大，摩擦系数较小的情形.

当设定四个长方体水平方向上随机的偏移程度为零时，它们最终会呈现上下堆叠的状态. 测试呈现出，在恢复系数较大时，前面依次计算冲量更新速度的方法会产生明显的上下震动，而约束动力学的实现直接解出全局的解，相对前者会更加稳定.
